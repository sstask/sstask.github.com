把hexo博客备份到github

上个blog讲了搭建hexo blog的方法，并且可以把博客放在github上，很简单而且很有效，github是个好地方。
但是github上保存的只是hexo生成好的html文件和一些博客用到的图片之类的，而我想github能保存我博客的源文件，就是.md文件。
刚开始我想已附件的方式把源文件链接到网站里面，这样github就会把他们当成资源文件保存了，可是hexo好像不支持附件链接。
hexo或者说markdown支持图片的链接，于是我想到了把我的博客源文件制作成图片然后链接进博客html里，这样就能达到保存的效果了。
直接讲我怎么做的吧：

  • 首先下载二维码生成工具

    yum install qrencode

  • 对源文件生成二维码图片

    qrencode -o my.md.png “my.md”

  • 在二维码图片的后面追加分隔符
  • 在二维码图片后面追加博客源文件的所以内容
  • 把二维码图片链接进博客
  • 生成并发布博客

这样我的博客源码就被写在了图片了，图片又被保存在了github上，再也不用担心博客的丢失了。
如果想还原博客的话就读取图片内存然后取出分隔符之后的全部内容就ok了。
整个流程还是有点小复杂的，所以我写了一个python脚本来完成，他可以完成整个发布博客和还原博客的过程。

linux搭建hexo博客并发布到github

    之前用了wrodpress做博客，自动化做的挺好的，省时省力，一键安装，而且有各种主题各种插件，都挺好的，唯一的不爽在于内部实现挺复杂，发布博客和打开博客的速度比较慢，估计和操作数据库有很大关系吧。于是在网上大概搜了下其他发布博客的框架，发现很多人推荐hexo，所以尝试了下，记录下安装过程，不是那么省心。

由于hexo是基于node.js的，所以要先安装node.js，在node.js官网下载。刚开始我下载的最新源码，于是开始编译，发现源码很多基于C++11的，但我的编译器不支持，还要升级gcc，折腾了下没有成功放弃。然后下载了node.js的0.10版本，编译且安装成功，但在安装hexo的时候提示需要0.12版本以上，于是又放弃。

最终我选择了下载node.js最新release版本(非源码，直接是可执行程序)，node-v4.0.0-linux-x64.tar.gz，解压，然后执行

ln -s /home/hexo/node-v4.0.0-linux-x64/bin/node /usr/local/bin/node
ln -s /home/hexo/node-v4.0.0-linux-x64/bin/npm /usr/local/bin/npm

（我创建的用户就是hexo，这个按你自己的来），这样node.js应该就能用了。

然后安装git，yum install git，最后就可以按照hexo官方介绍安装了，

如果成功，就可以在本地连接中浏览到默认博客了。

这是博客也只能在本地浏览，如果你想在外网能浏览就需要自己申请域名和外网机器，或者直接发布到github，github为每位用户提供了一个独有域名，可以通过这个域名直接访问博客。

首先申请github账户，比如用户名是xxx，然后创建一个代码仓库，命名为xxx.github.io。

在本地机器上运行git config –global user.email “xxx@xxx.com” git config –global user.name “xxx” 配置git，然后执行ssh-keygen -t rsa -C “xxx@xxx.com”（邮箱是申请github用的邮箱），复制~/.ssh/id_rsa.pub中的秘钥。

点击github上的 Account settings -> SSH Keys -> Add SSH Key，把复制的秘钥填进key里，这样本地机访问github就不需要密码了。
